# 团队能力测评产品 · 完整方案（v2）

> **一句话定位**：AI像经验丰富的队长一样跟你聊20分钟，聊完告诉你"你在队里该待在什么位置"，以及"你没意识到的自己"。

---

## 一、产品定位

### 核心差异

不是"把人工测评流程AI化"，而是**让AI发挥它独有的能力**：动态判断、主动发现、跨域联想、实时挑战。

| 对比维度 | ChatGPT / 通用AI | 旧方案（流程驱动） | 本方案（AI能动性驱动） |
|---------|-----------------|------------------|---------------------|
| 对话逻辑 | 用户问它答 | 固定四段式，每段干什么写死 | AI自主决定聊什么、深挖什么、何时切换 |
| 软实力测法 | 直接问"你抗压能力如何" | 固定问2个关键事件 | AI根据对话走向自然引出，不预设 |
| 意外发现 | 不会 | 不会——流程之外的信息被忽略 | **会——捕捉用户无意间暴露的能力点，主动深挖** |
| 矛盾追问 | 不会 | 不会——流程不允许跑偏 | **会——发现自评与表现矛盾时主动指出** |
| 挑战用户 | 不会，只会迎合 | 不会——怕用户不爽 | **会——有理有据地挑战用户的自我认知** |
| 画像生成 | 最后一次性输出 | 最后按模板输出 | **边聊边输出判断，最后整合成画像** |

> **旧方案的问题是：把AI当成了一个"会说话的问卷"。AI真正的价值不在于执行流程，而在于流程之外的那些判断。**

### 双视角设计

| 角色 | 看到什么 | 核心价值 |
|------|---------|---------|
| 组织者 | 全队能力矩阵 + 协作风格图谱 + 缺口诊断 + 分工建议 | 知道谁干什么、还缺谁、谁搭谁 |
| 队员 | 个人能力定位卡 + 行为模式卡 + 自我认知校准 | 知道自己擅长啥、盲区在哪、在队里是什么角色 |

### 对个人"厚度"的三层结构

```
第一层：你会什么              ← 技能测评
        "我数据分析8分"        事实层，最薄

第二层：你怎么做事            ← 行为模式推断
        "我压力下倾向自己扛"   模式层，中等厚度

第三层：你不了解自己的什么    ← 自评-实测偏差 + AI主动发现
        "我以为自己PPT不行，   认知层，最厚
         其实做得还不错"
```

## 二、能力维度：贴合项目书实际结构

不再用通用的6维度，直接对齐项目书的5个组别。**测评结果可以直接指导分工。**

| 组别 | 核心能力 | 项目书对应板块 | 要什么作品验证 |
|------|---------|--------------|--------------|
| 📊 背景+市场组 | 行业调研、市场分析、竞品分析、文献综述、政策解读 | 项目背景、行业分析、市场分析、竞品分析 | 调研报告、市场分析文档、文献综述 |
| 🎯 产品组 | 需求洞察、方案设计、用户研究、产品思维、原型制作 | 产品方案、功能设计、用户分析 | 产品方案文档、原型图、用户调研报告 |
| 💻 技术组 | 技术方案设计、开发实现、架构理解、技术选型 | 技术方案、实现路径、技术可行性 | 代码、技术方案文档、系统架构图 |
| 💰 财务组 | 成本核算、财务预测、商业模式设计、融资规划 | 财务预测、商业模式、成本分析 | 财务预测表、商业模式画布、成本分析 |
| 🎨 美工组 | 视觉设计、PPT设计、UI设计、品牌物料 | 视觉呈现、路演物料、UI设计 | PPT、设计稿、UI界面 |

**重要设计**：一个人不一定只属于一个组。AI判断的是"你在5个组别上的能力分布"，而不是"你属于哪个组"。一个技术很强但产品思维也不错的人，AI会建议他"技术主攻 + 参与产品讨论"。

### 各组别的评分标准

#### 📊 背景+市场组

| 评分项 | 分值 | 看什么 |
|--------|:----:|--------|
| 数据来源扎实度 | 3 | 是有一手数据/权威报告，还是拍脑袋/百度随便搜的 |
| 分析框架清晰度 | 3 | 有没有结构化分析（PEST/SWOT/波特五力等），还是流水账 |
| 洞察深度 | 2 | 是罗列信息还是有自己的判断和发现 |
| 竞品分析意识 | 2 | 有没有对比意识，知不知道对手在做什么 |

#### 🎯 产品组

| 评分项 | 分值 | 看什么 |
|--------|:----:|--------|
| 需求理解 | 3 | 是从真需求出发，还是自嗨式设计 |
| 方案逻辑 | 3 | 功能设计有没有内在逻辑，不是功能堆砌 |
| 用户视角 | 2 | 能不能站在用户角度想问题 |
| 落地意识 | 2 | 方案能不能实现，有没有考虑资源约束 |

#### 💻 技术组

| 评分项 | 分值 | 看什么 |
|--------|:----:|--------|
| 方案合理性 | 3 | 技术选型是否匹配需求，有没有过度设计或设计不足 |
| 实现复杂度 | 3 | 实现了什么、代码量、功能完整度 |
| 工程意识 | 2 | 模块化、错误处理、版本管理、文档 |
| 技术表达 | 2 | 能不能把技术方案给非技术的人讲明白 |

#### 💰 财务组

| 评分项 | 分值 | 看什么 |
|--------|:----:|--------|
| 逻辑自洽 | 3 | 收入模型、成本结构、增长率之间是否自洽 |
| 数据有据 | 3 | 预测数据有没有依据，还是随便填的 |
| 商业敏感度 | 2 | 对商业模式的理解，知不知道钱从哪来 |
| 风险意识 | 2 | 有没有考虑财务风险和应对方案 |

#### 🎨 美工组

| 评分项 | 分值 | 看什么 |
|--------|:----:|--------|
| 视觉审美 | 3 | 配色、排版、字体是否协调美观 |
| 信息传达 | 3 | 核心信息是否一眼能get，不是华而不实 |
| 设计一致性 | 2 | 风格、字体、配色是否统一 |
| 效率意识 | 2 | 有没有模板意识、组件复用，还是每页从零做 |

## 三、核心范式：AI能动性驱动的对话

### 旧方案的问题

旧方案是"固定四段式"——开场锚定→深挖Top2→探测盲区→生成画像。每一步干什么、花多长时间、问什么都写死了。

这本质上是**把AI当成了会说话的问卷**。流程之外的任何信息都被忽略了，AI没有自主判断的空间。

### 新范式：松框架 + AI自主判断

保留一个**松框架**作为时间约束，但框架内完全由AI自主决定：

```
开场（2-3min）  → AI听用户自我介绍，形成初步判断
  ↓
探索（12-14min）→ AI自主决定聊什么、深挖什么、何时切换
  ↓                自由探索5个组别 + 自然引出关键事件
收束（3-4min）  → AI整合所有判断，生成画像
```

**框架只约束三件事**：
1. 总时长约20分钟
2. 5个组别都要覆盖到（深浅由AI判断）
3. 至少引出1个关键事件（时机由AI把握）

**框架之外，全部由AI自主决定**：
- 先聊哪个组别
- 每个组别深挖到什么程度
- 什么时候切换话题
- 什么时候插入关键事件问题
- 什么时候挑战用户
- 什么时候给反馈

---

### AI的6种能动性行为

这是本方案和旧方案、和ChatGPT的**核心区别**。

#### 行为1：动态锚定——不按固定顺序，按用户特征决定方向

AI不是机械地"先聊Top1再聊Top2"，而是根据用户开场回答的特征，动态决定最有价值的探索方向。

**示例**：

> 用户："我比较擅长数据分析，做过SPSS回归。也写过文献综述。PPT的话也能做。"
>
> **旧方案的AI**：按顺序深挖数据分析→文献综述→PPT，每个走固定流程。
>
> **新方案的AI**：注意到用户描述数据分析时说得很具体（SPSS回归），但描述PPT时只有一句"也能做"——这暗示PPT不是核心能力。AI决定：深挖数据分析，快速扫一下文献综述，PPT只需要一个轻问题确认。**省下的时间用来探索用户没提到但可能擅长的方向**。

#### 行为2：意外发现——捕捉用户无意间暴露的能力点

用户在描述一件事时，可能无意间暴露了另一个能力。AI要能捕捉到，并主动深挖。

**示例**：

> 用户描述数据分析经历时说："我当时还顺手做了个可视化dashboard给团队用，大家觉得挺方便的。"
>
> **旧方案的AI**：忽略这句话，继续走数据分析的评分流程。
>
> **新方案的AI**：捕捉到"可视化dashboard"——这可能暗示产品组的能力。AI标记这一点，在后续对话中主动探索："你刚才说做了个dashboard，你是怎么决定放哪些数据的？有没有想过用户会怎么用？"**这是固定流程永远覆盖不到的发现**。

#### 行为3：矛盾追问——发现自评与表现矛盾时主动指出

AI不回避矛盾，而是把矛盾变成洞察。

**示例**：

> 用户开场说"我团队协作能力很强"，但描述所有项目时都用"我做了XX"，从没提到队友。
>
> **旧方案的AI**：不处理这个矛盾。
>
> **新方案的AI**："我注意到一个有意思的事——你说自己团队协作能力强，但刚才描述项目时都是'我做了XX'，没怎么提到队友。你能聊聊你在团队里是怎么和别人配合的吗？还是说你更习惯一个人干活？"**这个追问本身就是一次行为模式测评**。

#### 行为4：实时重塑判断——发现意外天赋时主动调整方向

聊到一半，AI发现用户在某个方向表现出意外天赋，主动调整后续对话重心。

**示例**：

> 用户自评"技术组"，但聊技术方案时，AI发现他描述技术的方式特别有逻辑、特别能站在用户角度想问题。
>
> **旧方案的AI**：继续走技术组评分流程。
>
> **新方案的AI**："你描述技术方案的方式很清楚，连我这种非技术的都能听懂。你有没有想过往产品方向试试？你刚才说功能X是为了解决用户痛点Y——这个思路其实很产品。你平时会主动想这些吗？"**AI主动拓宽用户的自我认知边界**。

#### 行为5：主动联想——基于对话内容做跨组别推荐

AI不只测用户说自己擅长的，还主动联想"你可能还擅长XX"。

**示例**：

> 用户在背景+市场组表现不错，描述竞品分析时条理清晰、数据扎实。
>
> **新方案的AI**："你对数据的敏感度和分析框架的意识，其实在做财务预测时也很有用。很多财务分析的核心就是结构化地拆解数据。你有没有接触过财务建模？"**跨组别联想，帮用户发现自己的潜在能力**。

#### 行为6：挑战自评——有理有据地挑战用户的自我认知

AI不只迎合用户，在有明显证据时会主动挑战。

**示例**：

> 用户说"我PPT做得不好"，但AI在对话中注意到用户描述事物时经常用视觉化比喻，表达很有画面感。
>
> **新方案的AI**："你说自己PPT不好，但你描述问题的方式其实很有画面感——你刚才说'让数据像讲故事一样流出来'，这个表达本身就说明你有视觉化思维。你确定是PPT做得不好，还是只是没找到合适的方法和模板？"**不是否定用户，而是帮用户重新审视自己的判断**。

---

### 同一个用户，AI可能走出完全不同的对话路径

这才是"AI能动性"的真正体现——**不是预设路径，而是实时生长的对话**。

以一个用户为例，假设他说"我擅长数据分析、也做过市场调研、PPT能做"：

```
路径A（AI判断数据分析值得深挖）：
  开场 → 深挖数据分析（发现方法扎实）→ 顺手发现产品思维 →
  快速扫市场调研 → 轻问PPT → 关键事件 → 画像

路径B（AI判断市场调研描述更有意思）：
  开场 → 先聊市场调研（发现竞品分析能力强）→ 联想到财务潜力 →
  回到数据分析快速确认 → PPT轻问 → 关键事件 → 画像

路径C（AI在聊数据分析时发现矛盾）：
  开场 → 聊数据分析 → 发现用户说"我做了"但描述含糊 →
  矛盾追问 → 用户坦承是队友做的 → 调整判断 →
  探索其他方向 → 关键事件 → 画像
```

> **旧方案：同一个用户走同一条路径。新方案：AI根据每句话的判断，实时决定下一步。**

## 四、对话过程中的关键设计

### 开场（2-3分钟）

**AI开场话术**（轻松、有温度，不像考试）：

> "嗨，先随便聊聊——你觉得自己比较擅长的3件事是什么？可以是任何方面，比如做过的调研、写过的东西、写过的代码、做过的PPT都行。每件事最好带一句你做过的例子，不用很长。"

**AI在开场阶段要做的事**：

1. 提取用户自述的能力标签，映射到5个组别
2. **记录自评分数**（为后续偏差分析做准备）
3. 判断每个标签的描述具体度（决定深挖优先级）
4. 形成初步的探索计划——**但随时准备推翻它**

**即时反馈**（开场就要给，不能让用户觉得在"被审"）：

> "听起来你在市场调研和数据分析上经验比较多。我接下来想重点聊聊这两个，顺便也看看其他方面。对了，你刚才提到做市场调研时'用了竞品对比'——这个挺有意思的，待会儿我想多问两句。"

### 探索阶段（12-14分钟）

这是AI能动性发挥的主要阶段。AI在这个阶段：

**要做的事**：
- 按自主判断的优先级，逐个探索5个组别
- 每个组别至少问到"有没有做过+做到什么程度"
- 对用户擅长的方向深挖到细节层
- 有作品就问要不要传（可选，不卡流程）
- 自然地引出1-2个关键事件
- 实时给出判断反馈
- 主动执行6种能动性行为

**作品验证的话术**（不给压力）：

> "听起来做得挺扎实的。如果你手边有当时的报告、截图、代码什么的，可以传给我看看，我能给更准的评价。不方便也没事，咱们接着聊。"

**关键事件的引出时机**（不是固定在某个位置，而是自然引出）：

| 触发条件 | AI怎么引出 |
|---------|-----------|
| 用户提到项目压力大 | "你说那次时间很紧——当时最崩溃的是哪个环节？怎么扛过来的？" |
| 用户提到团队合作 | "你提到和队友分工——有没有过意见严重不合的时候？怎么解决的？" |
| 用户提到失败经历 | "这次没做成，你当时第一反应是什么？是立刻想补救还是先缓一下？" |
| 对话快结束时还没自然引出 | "差不多聊完了，最后想听你聊一个具体的事——做项目时最崩溃的一次是什么情况？" |

**行为痕迹收集**（贯穿全程，用户无感知）：

| 痕迹 | 推断什么 |
|------|---------|
| 描述项目时说"我做了"还是"我们做了" | 团队意识 vs 个人主义 |
| 面对追问时是坦然回答还是开始含糊 | 真实度、自信来源 |
| 传作品的态度（爽快/犹豫/找借口不传） | 配合度、开放度 |
| 自评分数和实际表现的差距 | 自我认知校准度 |
| 语言风格（结构化表达 vs 意识流） | 沟通风格、思维条理性 |
| 被质疑时的反应（接受/辩解/反思） | 成长型 vs 固定型思维 |
| 描述困难时是否提及他人 | 求助倾向 vs 独立扛 |

### 收束阶段（3-4分钟）

**AI判断对话可以收束的信号**：
- 5个组别都已覆盖
- 至少引出1个关键事件
- 对用户的优势和盲区有了足够判断

**收束话术**：

> "好了，聊得差不多了。根据你说的这些，我觉得你在团队里比较像是一个XX型的角色。不过有几个地方跟你自己说的不太一样，挺有意思的。我给你出一张能力定位卡，你看看准不准。"

**四个关键设计原则**：

1. **每个结论都有证据**——不说"你市场分析能力强"，说"证据：对话中能用PEST拆解行业……"
2. **区分三种状态**——✅已验证 / ⚠️待验证 / ❓未涉及
3. **给模式不给标签**——不说"你是行动型人格"，说"在压力下你倾向于先行动"
4. **主动声明盲区**——把测不到的维度列出来，管理预期

**新增"AI主动发现的"板块**——这是旧方案没有的，体现AI能动性的核心价值：不只是验证用户说的，而是**告诉用户他没说的**。

## 六、MVP落地路径

### 第1步：扣子Bot验证（1-2天）

用扣子搭一个Bot：

1. **System Prompt**写入：
   - 5组别评分标准
   - 6种AI能动性行为的具体指引
   - 行为痕迹收集清单
   - 关键事件引出逻辑
   - 画像生成模板
2. **人设**：不是"测评官"，而是一个"经验丰富的学长/队长"——有判断力、会说真话、但不咄咄逼人
3. **关键指令**：明确告诉AI"不要按固定顺序问，根据用户回答自主决定下一步"

**验证目标**：找3-5个人测，看四件事——
- 画像和真实的人对不对得上？（准不准）
- 20分钟能不能跑完？中途有没有想退出？（体验好不好）
- AI有没有做出"意外发现"或"矛盾追问"？（能动性有没有发挥）
- 行为模式推断有没有让人觉得"被看见"？（厚不厚）

### 第2步：补充团队看板（3-5天）

Bot跑通后，做一个简单的前端页面：
- 组织者创建团队，生成邀请链接
- 队员做完测评，画像自动归入团队
- 组织者看到团队矩阵 + 协作风格图谱 + 诊断报告

### 第3步：迭代优化

根据测试数据调整：
- AI的能动性行为有没有发挥出来？（如果还是走固定流程，说明prompt要改）
- 哪些组别的追问最有效？
- 画像的"准不准"反馈率多少？
- "AI主动发现的"板块命中率多少？

## 七、发展路径

```
MVP（现在）
  → 5组别测评 + AI能动性驱动 + 行为痕迹推断
  → 输出技能画像 + 行为模式卡 + 自我认知校准 + AI主动发现
  → 扣子Bot验证，3-5人测试

V2（3个月后）
  → 增加"项目后复盘"功能：项目结束后让队友互评
  → 对比自评/实测/他评三方数据，校准更准
  → 行为模式卡从"推断"升级为"多方验证"

V3（6个月后）
  → 积累足够数据后，做"行为模式成长轨迹"
  → "你半年前倾向自己扛，最近开始学会求助了"
  → 画像从静态快照升级为动态成长曲线

V4（长期）
  → 从比赛场景迁移到工作场景
  → 项目预判能力：基于团队能力配置，预判项目风险
  → "你们队缺美工能力，如果路演占比高，
     有70%概率在物料上失分"
```

## 八、风险提示与应对

| 风险 | 应对 |
|------|------|
| 用户嫌20分钟太长 | 关键事件可压缩到1个，总时长压到17分钟 |
| 作品上传体验差 | 强调"可选"，不传不影响流程 |
| 画像不准 | 主动声明盲区 + 用户反馈机制 |
| AI能动性发挥不出来（又变回固定流程） | System Prompt中明确禁止"按固定顺序问"，并给出6种能动性行为的具体场景示例 |
| AI过度挑战用户导致体验差 | 挑战必须有依据，且用"提问"而非"否定"的方式 |
| 被ChatGPT复制 | 核心壁垒在AI能动性 + 团队看板 + 行为痕迹分析，不在单点测评 |

## 九、一句话总结

> **旧方案把AI当成了"会说话的问卷"——流程写死，AI只在框架内填内容。新方案把AI当成"有判断力的队长"——只给时间约束和必须覆盖的维度，剩下全由AI自主决定。AI能动性的6种行为（动态锚定、意外发现、矛盾追问、实时重塑、主动联想、挑战自评）才是和ChatGPT的本质区别。能力维度对齐项目书5个组别，测评结果直接指导分工。**
